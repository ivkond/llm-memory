<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Shell hook → MCP server | Untrusted input (project path) crosses here via curl |
| Hook script execution | Claude Code runs arbitrary shell commands |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | Hook scripts | mitigate | Validate MCP_PORT is localhost only, reject remote URLs |
| T-05-02 | Repudiation | Session summary | accept | Script logs to stdout; user can inspect transcript |
| T-05-03 | Information Disclosure | Context injection | mitigate | Token budget limits data exposed per session |
| T-05-04 | Denial of Service | MCP server down | accept | Hooks fail silently;不影响Claude继续工作 |
| T-05-05 | Elevation of Privilege | Shell access | accept | Hooks run as user who started Claude Code |

**Note:** This phase primarily integrates with an already-trusted MCP server running on loopback. The main risk is script tampering, mitigated by keeping hooks in user's .claude directory with standard file permissions.
</threat_model>
