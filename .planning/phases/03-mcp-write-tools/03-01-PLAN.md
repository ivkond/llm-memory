<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → wiki_remember_fact | Untrusted input (content string) crosses into RememberService + SanitizationService |
| MCP client → wiki_ingest | Untrusted input (source URL/path) crosses into IngestService + LLM extraction |
| MCP client → wiki_lint | Controlled input (phase selection), safe operation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Injection | wiki_remember_fact | mitigate | SanitizationService with REDACT mode strips secrets before storage. Input validated by RememberService (ContentEmptyError). |
| T-03-02 | Injection | wiki_ingest | mitigate | IngestService.validateTargetPath() validates LLM-provided paths before filesystem write (blocks path traversal, enforces wiki/projects/ prefix). Source size capped at 100K tokens. |
| T-03-03 | Denial of Service | wiki_ingest | mitigate | Worktree isolation ensures main branch is never touched on failure. MAX_SOURCE_TOKENS guard before worktree creation. |
| T-03-04 | Information Disclosure | All write tools | accept | Response envelopes contain only service-generated paths/IDs, not internal state. |

</threat_model>
